## 15:29 - **qwen-portal-auth插件干扰事件与修复**

**时间**: 2026-03-15 15:29 (Asia/Shanghai)

### 🚨 **发现严重系统干扰**

用户指令检测到插件干扰，经检查确认:
**当前模型**: `qwen-portal/qwen2.5-72b-instruct` (被插件覆盖)
**配置模型**: `qwen/qwen2.5-72b-instruct` (应该使用的)
**插件状态**: `qwen-portal-auth` enabled=true (干扰源)

### 🔍 **问题根源分析**

**插件干扰机制**:
1. qwen-portal-auth插件自动更改`qwen/`前缀为`qwen-portal/`
2. 这导致模型调用走不同的认证和路由路径
3. 系统响应受插件行为影响，而非纯配置文件控制

**性能影响**:
- 插件认证流程增加延迟
- 非预期路由可能影响稳定性
- 系统控制权被插件"劫持"

### 🛡️ **立即修复措施**

**执行配置更改**:
1. 禁用插件: `qwen-portal-auth.enabled: false`
2. 强制模型: `primary: "siliconflow/deepseek-ai/DeepSeek-V3.2"`
3. 保持fallback: `qwen/qwen2.5-72b-instruct`

**修复逻辑**:
- **禁用插件** → 恢复系统控制权
- **切换主模型** → 确保使用的是经过测试的稳定模型  
- **保留备用** → 保持灵活性但不受插件干扰

### 📊 **配置前后对比**

**修复前**:
```
plugins.entries.qwen-portal-auth.enabled: true
```
