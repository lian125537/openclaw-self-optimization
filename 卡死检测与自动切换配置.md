# 卡死检测与自动切换 DeepSeek 模型配置

## 配置说明

OpenClaw 已经配置了故障转移机制，当小米模型出现卡死、超时或其他错误时，会自动切换到 DeepSeek 模型。

## 当前配置状态

### ✅ 故障转移配置

```json
"model": {
  "primary": "xiaomi/mimo-v2-flash",
  "fallbacks": ["deepseek/deepseek-chat"]
}
```

### ✅ 触发条件

当小米模型遇到以下错误时，会自动切换到 DeepSeek：

1. **超时（timeout）** - 请求超时
2. **认证失败（auth）** - API 密钥无效或过期
3. **速率限制（rate_limit）** - 请求频率过高
4. **配额不足（quota）** - API 配额用尽

## 卡死检测机制

### 1. 超时检测

OpenClaw 会检测模型请求的超时情况：

- **默认超时时间**: 600 秒（10分钟）
- **超时触发故障转移**: 当请求超时时，会自动切换到备用模型

### 2. 卡死现象检测

卡死现象通常表现为：

- **请求无响应**: 模型长时间不返回结果
- **连接超时**: 网络连接超时
- **处理超时**: 模型处理时间过长

OpenClaw 会将这些情况分类为超时错误，并触发故障转移。

## 配置优化建议

### 1. 调整超时时间（可选）

如果需要调整超时时间，可以在配置文件中添加：

```json
"agents": {
  "defaults": {
    "timeoutSeconds": 300,  // 5分钟超时
    "model": {
      "primary": "xiaomi/mimo-v2-flash",
      "fallbacks": ["deepseek/deepseek-chat"]
    }
  }
}
```

### 2. 添加更多备用模型（可选）

如果需要更多备用模型，可以扩展故障转移列表：

```json
"model": {
  "primary": "xiaomi/mimo-v2-flash",
  "fallbacks": [
    "deepseek/deepseek-chat",
    "openai/gpt-4o-mini"
  ]
}
```

## 工作流程

### 正常情况

```
用户请求 → 小米模型 (xiaomi/mimo-v2-flash) → 返回响应
```

### 卡死/超时情况

```
用户请求 → 小米模型 (xiaomi/mimo-v2-flash)
    ↓ (卡死/超时)
DeepSeek 模型 (deepseek/deepseek-chat) → 返回响应
```

## 相关配置

### 超时配置

- **默认超时**: 600 秒（10分钟）
- **可配置位置**: `agents.defaults.timeoutSeconds`
- **触发故障转移**: 超时错误会触发模型切换

### 故障转移配置

- **主模型**: `xiaomi/mimo-v2-flash`
- **备用模型**: `deepseek/deepseek-chat`
- **触发条件**: 超时、认证失败、速率限制、配额不足

## 注意事项

1. **超时时间**: 默认 10 分钟，可根据需要调整
2. **故障转移日志**: 可在 OpenClaw 日志中查看切换信息
3. **模型性能**: DeepSeek 模型响应速度可能有所不同
4. **费用控制**: 注意监控 API 调用次数和费用
