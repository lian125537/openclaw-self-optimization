OpenClaw是一个开源的AI助手平台，让你可以拥有自己的AI助手。

#### 1.2 核心组件
- **网关**: 核心服务，管理所有功能

- **模型**: AI大脑，处理对话和任务
- **技能**: 扩展功能，如天气查询、文件处理
- **工具**: 系统操作能力，如文件读写、命令执行

### 第二章：安装与配置

#### 2.1 系统要求
- 操作系统: Windows 10/11, macOS, Linux
- 内存: 至少4GB RAM
- 存储: 至少1GB可用空间
- 网络: 互联网连接（用于API调用）

#### 2.2 安装步骤
1. 安装Node.js (v16或更高版本)
2. 安装OpenClaw: `npm install -g openclaw`
3. 初始化配置: `openclaw init`
4. 启动服务: `openclaw gateway start`

#### 2.3 基本配置
编辑 `~/.openclaw/config.json`：
```json
{
  "模型": {
    "默认模型": "你的模型",
    "API密钥": "你的密钥"
  },
  "界面": {
    "端口": 18789,
    "主题": "深色"
  }
}
```

#### 3.1 启动与停止

- 状态: `openclaw gateway status`

## 🛠️ 转换工具使用说明

### 使用方法
```powershell
# 试运行（不实际修改文件）
.\convert_to_chinese.ps1 -DryRun

# 实际转换（自动备份）
.\convert_to_chinese.ps1 -Backup

# 指定目录
.\convert_to_chinese.ps1 -OpenClawDir "C:\自定义路径\.openclaw" -Backup
```

### 转换范围
- ✅ JSON配置文件
- ✅ YAML配置文件  
- ✅ Markdown文档
- ✅ 文本文件
- ✅ JavaScript/TypeScript文件中的注释
- ✅ 配置文件中的键名和值

### 不转换的内容
- ❌ 二进制文件（exe、dll、图片等）
- ❌ 代码中的技术标识符（变量名、函数名）
- ❌ URL和API端点
- ❌ 数字和布尔值（仅转换显示文本）

## 📊 注意事项

### 技术限制
1. **键名转换**: JSON键名转换为中文后，代码可能无法正确解析
2. **API兼容性**: 某些API可能要求英文参数名
3. **插件兼容性**: 第三方插件可能不支持中文配置

### 解决方案
1. **混合配置**: 保持技术键名为英文，仅转换显示文本
2. **映射层**: 添加中英文映射层
3. **渐进式转换**: 逐步转换，分阶段实施

### 回滚方案
如果转换后出现问题：
```powershell
# 恢复备份
Remove-Item "$env:USERPROFILE\.openclaw" -Recurse -Force
Copy-Item "$env:USERPROFILE\.openclaw-backup" "$env:USERPROFILE\.openclaw" -Recurse
```

## 🎯 最佳实践建议

### 推荐方案：显示层中文化
保持底层配置为英文，仅在界面显示时转换为中文：

{
