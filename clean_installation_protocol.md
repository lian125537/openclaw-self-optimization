# 🍎 OpenClaw 史蒂夫·乔布斯级洁净安装协议

## 🚨 **当前状态诊断**
```
症状: Gateway启动后立即崩溃(无错误日志)
根本原因: 安装环境污染
影响: 成本控制失效、系统不稳定、600万Token异常
紧迫性: 高度紧急 (成本正在流失)
```

## 📋 **洁净安装步骤**

### 阶段1: 完全卸载 (不留任何残余)
```
1. 停止所有OpenClaw相关服务
2. 删除Windows Task Scheduler中的定时任务
3. 卸载npm全局包
4. 清除所有配置文件和目录
5. 清理临时文件和日志
```

### 阶段2: 环境净化
```
1. 检查Node.js版本兼容性 (25.2.1需要降级)
2. 清除可能导致崩溃的代理设置
3. 验证网络端口状态
4. 检查系统权限问题
```

### 阶段3: 洁净安装
```
1. 使用干净的网络环境重新安装
2. 最小化依赖配置
3. 分阶段验证各组件
4. 启用详细日志记录
```

### 阶段4: 验证与恢复
```
1. 验证Gateway稳定运行
2. 恢复重要的配置设置
3. 重新启用监控系统
4. 启用成本控制防护
```

## 🔐 **关键风险与防护措施**

### 🚫 **成本控制防护**(立即启用)
1. **安装过程中禁用所有AI功能**
2. **配置最大Token限制**(10,000/会话)
3. **启用成本告警**(超过$0.50立即警报)
4. **备用免费模型**(切换到Ollama/本地模型)

## 🎯 **乔布斯原则应用**

### 🍏 **简单性 > 复杂性**
- 最小化初始配置
- 只启用核心功能
- 分阶段增加复杂度

### 🌀 **现实扭曲场**
- 创造"Just Works"的体验
- 隐藏复杂性，提供简洁界面
- 让成本控制自动且无感

### 🔄 **迭代完美**
- 先安装核心(Gateway)
- 验证2小时后添加Agent
- 再验证2小时后添加定时任务
- 最后添加复杂技能
