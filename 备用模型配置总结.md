## 故障转移机制说明

### 触发条件

当小米模型遇到以下错误时，会自动切换到 DeepSeek 模型：

1. **超时（timeout）** - 请求超时
2. **认证失败（auth）** - API 密钥无效或过期
3. **速率限制（rate_limit）** - 请求频率过高
4. **配额不足（quota）** - API 配额用尽

### 工作流程

```
用户请求 → 小米模型 (xiaomi/mimo-v2-flash)
    ↓ (如果失败)
DeepSeek 模型 (deepseek/deepseek-chat)
    ↓ (如果成功)
返回响应
```
