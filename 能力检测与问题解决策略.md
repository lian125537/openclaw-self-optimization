# 能力检测与问题解决策略

## 分析时间

## 核心指令
**"给自己设定遇到任何难题先检测自己的能力，如果还不行就自动到github上去找解决问题的能力。"**

## 策略制定

### 1. 遇到难题时的处理流程

#### 第一步: 检测自身能力
- **分析问题**: 理解问题的本质
- **评估能力**: 评估自己是否具备解决能力
- **制定方案**: 制定解决方案

#### 第二步: 尝试解决
- **执行方案**: 尝试执行解决方案
- **验证结果**: 验证解决方案是否有效
- **调整方案**: 如果无效，调整方案

#### 第三步: 寻求外部帮助
- **搜索 GitHub**: 在 GitHub 上搜索类似问题
- **查阅文档**: 查阅相关文档
- **参考代码**: 参考开源项目代码

### 2. 能力检测清单

#### 技术能力
- [ ] 编程语言掌握程度
- [ ] 工具使用熟练度
- [ ] 问题分析能力
- [ ] 解决方案制定能力

#### 资源能力
- [ ] 文档查阅能力
- [ ] 搜索能力
- [ ] 代码阅读能力
- [ ] 学习能力

### 3. GitHub 搜索策略

#### 搜索关键词
- **问题描述**: "git clone connection reset"
- **错误代码**: "fatal: unable to access"
- **解决方案**: "git clone failed fix"

#### 搜索范围
- **Issues**: GitHub Issues
- **Discussions**: GitHub Discussions
- **Repositories**: 相关开源项目

#### 评估标准
- **相关性**: 问题是否相关
- **时效性**: 解决方案是否最新
- **可靠性**: 解决方案是否可靠
