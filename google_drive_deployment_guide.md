## 🚀 详细部署步骤

### 阶段1：创建Google Cloud项目（10分钟）

#### 步骤1.1：访问Google Cloud控制台
1. 打开浏览器，访问：**https://console.cloud.google.com**
2. 使用你的Google账户登录

#### 步骤1.2：创建新项目
1. 点击页面顶部的项目下拉菜单
2. 点击"新建项目"
3. 输入项目名称：**`openclaw-memory-system`**
4. 项目ID：自动生成即可
5. 点击"创建"

#### 步骤1.3：启用计费（如果需要）
1. 如果提示启用计费，点击"启用"
2. 选择付款方式（Google Drive 2TB需要付费）
3. 完成计费设置

### 阶段2：启用Google Drive API（5分钟）

#### 步骤2.1：进入API库
1. 在左侧菜单点击"API和服务" → "库"
2. 在搜索框中输入：**`Google Drive API`**
3. 点击搜索结果中的"Google Drive API"

#### 步骤2.2：启用API
1. 点击"启用"按钮
2. 等待API启用完成

### 阶段3：配置OAuth同意屏幕（10分钟）

#### 步骤3.1：创建OAuth同意屏幕
1. 在左侧菜单点击"API和服务" → "OAuth同意屏幕"
2. 选择用户类型：**`外部`**
3. 点击"创建"

#### 步骤3.2：填写应用信息
```
应用名称：OpenClaw Memory System
用户支持邮箱：你的邮箱
开发者联系信息：你的邮箱
```

#### 步骤3.3：添加范围
1. 点击"添加或删除范围"
2. 手动添加以下范围：
   - `https://www.googleapis.com/auth/drive.file`
   - `https://www.googleapis.com/auth/drive.metadata`
3. 点击"更新" → "保存并继续"

#### 步骤3.4：添加测试用户
1. 在"测试用户"部分
2. 点击"添加用户"
3. 输入你的Google邮箱地址
4. 点击"保存并继续"

### 阶段4：创建OAuth 2.0凭据（5分钟）

#### 步骤4.1：创建凭据
1. 在左侧菜单点击"API和服务" → "凭据"
2. 点击"创建凭据" → "OAuth 2.0 客户端ID"

#### 步骤4.2：配置应用类型
```
应用类型：桌面应用
名称：OpenClaw Desktop Client
```

#### 步骤4.3：保存凭据
1. 点击"创建"
2. **重要**：复制显示的**客户端ID**和**客户端密钥**
3. 点击"确定"

### 阶段5：获取授权代码（5分钟）

#### 步骤5.1：生成授权URL
我会为你生成一个授权URL，你需要：
1. 在浏览器中打开这个URL
2. 登录你的Google账户（如果未登录）
3. 同意应用访问权限
4. 复制返回的授权代码

#### 步骤5.2：授权流程
```
流程：
你的浏览器 → Google登录 → 同意权限 → 返回授权码 → 粘贴给我
```

### 阶段6：本地配置（我可以完成）

#### 步骤6.1：创建配置文件
```json
{
  "google_drive": {
    "client_id": "你复制的客户端ID",
    "client_secret": "你复制的客户端密钥",
    "redirect_uri": "http://localhost:8080/oauth2callback"
  }
}
```

#### 步骤6.2：实现认证流程
我会编写代码处理：
- OAuth令牌获取

### 阶段7：测试和验证（5分钟）

#### 步骤7.1：测试连接
1. 运行测试脚本
2. 验证能访问Google Drive
3. 测试文件上传/下载

#### 步骤7.2：验证功能
