# Session Memory Bridge 技能

## 概述
连接会话记忆和长期记忆的桥梁系统，实现三层记忆架构。

## 三层记忆体系

### 热层 (Hot Layer)
- **存储**: 当前会话上下文
- **特点**: 即用即丢，短期记忆
- **容量**: 有限（当前对话窗口）
- **用途**: 维持对话连贯性

### 温层 (Warm Layer)  
- **存储**: MEMORY.md + 每日记忆文件
- **特点**: 启动时加载，跨会话持久
- **容量**: 中等（关键事实和决策）
- **用途**: 保存重要信息和偏好

### 冷层 (Cold Layer)
- **存储**: 向量数据库 + 河流缓存
- **特点**: 语义搜索，无限扩展
- **容量**: 无限（所有历史记录）
- **用途**: 长期记忆和知识检索

## 河流缓存系统
- **原理**: 记忆像河流一样流动（小溪→河流→大海）
- **更新**: 每10分钟自动更新 `memory/RIVER_CACHE.md`
- **内容**: 最近3天时间线 + 话题相关精华
- **启动**: 直接读取缓存，无需等待注入

## 核心功能

### 1. 记忆注入
- 对话内容自动追加到冷层
- 重要信息提炼到温层
- 实时更新河流缓存

### 2. 记忆检索
- 语义搜索历史对话
- 相关话题自动关联
- 上下文智能补充

### 3. 记忆管理
- 定期清理过期记忆
- 重要性分级存储
- 压缩和优化存储

## 技术实现

### 混合 Embedding 策略
1. **优先**: SiliconFlow API（无资源占用）
2. **回退**: 本地 BGE-M3（离线可用）
3. **维度**: 1024

### 河流实时追加
```bash
python river_realtime.py --append "对话内容" --role user
```

### 启动加载顺序
```
SOUL.md → USER.md → MEMORY.md → memory/YYYY-MM-DD.md → 河流缓存 → 对话
```

## 配置
- **冷层数据库**: ChromaDB
- **Embedding模型**: SiliconFlow/BGE-M3
- **缓存更新**: Cron 每10分钟
- **记忆保留**: 最近30天详细，更早的摘要
